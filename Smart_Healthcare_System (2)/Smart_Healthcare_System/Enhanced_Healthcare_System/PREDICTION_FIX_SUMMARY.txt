PROBLEM:
═══════════════════════════════════════════════════════════════════════════════
Always predicted "Kidney Infection or Stone" with 89% risk, regardless of
which symptoms were selected.


ROOT CAUSE:
═══════════════════════════════════════════════════════════════════════════════
The DataFrame columns were not in the correct order that the ML model was
trained with. Python dictionaries don't guarantee order, so symptoms were
being sent to the model in random order, causing incorrect predictions.


SOLUTION:
═══════════════════════════════════════════════════════════════════════════════
Changed from using a dictionary to using a list with explicit column order:

BEFORE (Wrong):
    symptoms = {
        'bodypain': value1,
        'Hollow': value2,
        ...
    }
    df = pd.DataFrame([list(symptoms.values())], columns=list(symptoms.keys()))
    # Problem: Dictionary keys order is not guaranteed!

AFTER (Correct):
    symptom_values = [value1, value2, value3, ...]  # Explicit order
    column_names = ['bodypain', 'Hollow', 'cold and cough', ...]  # Exact order
    df = pd.DataFrame([symptom_values], columns=column_names)
    # Solution: Guaranteed correct order matching training data!
